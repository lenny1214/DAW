Control de Versiones con Git

Control de Versiones

Git permite a los desarrolladores llevar un registro de los cambios en su código fuente a lo largo del tiempo.

Distribuido

Es un sistema de control de versiones distribuido, lo que significa que cada desarrollador tiene una copia completa del historial del proyecto en su máquina local.

Repositorios:

En Git, un repositorio es una base de datos que almacena todos los cambios realizados en un proyecto. Hay repositorios locales (en la máquina de cada desarrollador) y repositorios remotos (en servidores en la nube o locales).

Ramificación (Branching):

Git permite la creación de ramas independientes del código, lo que facilita el desarrollo paralelo y la implementación de nuevas características sin afectar la rama principal (normalmente llamada master o main).

Fusionar (Merging):

Una vez que se completa el trabajo en una rama, los cambios pueden fusionarse con otras ramas, permitiendo integrar nuevas características o correcciones de errores en la rama principal.

Commit:

Un commit en Git es una unidad atómica de cambio. Representa una versión específica de los archivos en el repositorio en un momento dado. Cada commit tiene un identificador único (hash) que lo distingue de los demás..

GitHub y GitLab:

GitHub y GitLab son servicios en la nube que proporcionan funcionalidades adicionales para proyectos Git. Estos servicios facilitan la colaboración, el seguimiento de problemas, la revisión de código y la integración continua.

Comandos Básicos:

git init: Inicia un nuevo repositorio. git clone: Clona un repositorio existente. git add: Añade cambios al área de preparación (staging area). git commit: Guarda los cambios en el repositorio. git pull: Obtiene cambios desde un repositorio remoto. git push: Envía cambios a un repositorio remoto. git branch: Lista, crea o borra ramas. git merge: Fusiona ramas. git status: Muestra el estado de los archivos en el repositorio. git log: Muestra el historial de commits.

Git Stash:

Git Stash es una característica que permite a los desarrolladores guardar temporalmente cambios no comprometidos en un área de almacenamiento temporal (stash) para poder cambiar de rama o realizar otras operaciones sin comprometer esos cambios. git stash save: Guarda los cambios en un stash con un mensaje descriptivo. git stash list: Muestra la lista de stashes. git stash apply: Aplica el último stash y lo mantiene en la lista de stashes. git stash pop: Aplica y elimina el último stash. git stash branch: Crea una nueva rama y aplica el último stash en esa rama.
